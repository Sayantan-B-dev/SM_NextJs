# Routing Metadata

## Overview

The Metadata API in Next.js allows you to define metadata (title, description, etc.) for each page. This improves SEO and ensures content looks correct when shared or indexed by search engines. Next.js provides two ways to configure metadata:

- **Static metadata object** -- export a `metadata` constant
- **Dynamic `generateMetadata` function** -- export a function that returns metadata

## Where Metadata Can Be Defined

| File | Scope |
|------|-------|
| `layout.tsx` | Applies to all pages within that layout segment |
| `page.tsx` | Applies only to that specific page |

## Metadata Merging Rules

Metadata follows a top-down merge strategy:

1. Root layout metadata sets the base values
2. Nested layout metadata merges with and overrides parent values
3. Page metadata merges with and overrides layout values for matching properties
4. Properties not overridden by child routes are inherited from parent routes

## Static Metadata Object

### Root Layout Metadata

```tsx
// app/layout.tsx
import type { Metadata } from "next"

export const metadata: Metadata = {
  title: "Next.js",
  description: "Generated by Next.js",
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

### Page Metadata (Overrides)

```tsx
// app/about/page.tsx
import type { Metadata } from "next"

export const metadata: Metadata = {
  title: "About Code Evolution",
}

export default function About() {
  return <h1>About</h1>
}
```

### Resulting Metadata

| Page | title | description |
|------|-------|-------------|
| `/` | "Next.js" | "Generated by Next.js" |
| `/about` | "About Code Evolution" | "Generated by Next.js" |

The about page's title overrides the root layout's title, but the description is inherited because it was not redefined in the page.

## Dynamic Metadata with generateMetadata

Use `generateMetadata` when metadata depends on dynamic data (params, search params, fetched data):

```tsx
// app/products/[productId]/page.tsx
import type { Metadata } from "next"

type Props = {
  params: { productId: string }
}

export const generateMetadata = async ({
  params,
}: Props): Promise<Metadata> => {
  const title = await new Promise<string>((resolve) => {
    setTimeout(() => {
      resolve(`iPhone ${params.productId}`)
    }, 100)
  })

  return {
    title: `Product ${title}`,
  }
}

export default function ProductDetails({ params }: Props) {
  return <h1>Details about product {params.productId}</h1>
}
```

This is particularly useful for pages that display data from a database or external API.

## Static vs Dynamic Metadata

| Aspect | Static Metadata | Dynamic Metadata |
|--------|----------------|------------------|
| Declaration | `export const metadata` | `export async function generateMetadata()` |
| Data Source | Hardcoded values | Params, searchParams, fetch results |
| Use Case | Fixed pages (About, Contact) | Dynamic pages (Product details, Blog posts) |
| Async | No | Yes (returns Promise) |

## Metadata Inheritance Tree

```
app/layout.tsx          title: "My App", description: "App description"
  app/blog/layout.tsx   title: "Blog" (overrides title for /blog/*)
    app/blog/page.tsx   title: "Blog Home" (overrides title for /blog)
    app/blog/[slug]/page.tsx  (inherits "Blog" title from blog layout)
  app/about/page.tsx    title: "About Us" (overrides title for /about)
                        description: inherited from root layout
```

## Best Practices

- Always define a `title` and `description` in the root layout for a baseline
- Override `title` on each page for specific, descriptive titles
- Use `generateMetadata` for data-driven pages (products, posts, users)
- Keep descriptions unique per page for better SEO
- Use the `Metadata` type from `next` for TypeScript support
