# Layouts

## Overview

A layout is a UI component that is shared between multiple pages in your application. Think of it as a consistent wrapper -- such as a header and footer -- that appears on every page. Next.js makes layouts easy to implement using a special `layout.tsx` file.

## The Root Layout

Next.js automatically provides a root layout in `app/layout.tsx`. This layout is mandatory -- every Next.js application must have one. If you delete it, Next.js regenerates it automatically.

### Default Root Layout

```tsx
// app/layout.tsx
import type { Metadata } from "next"

export const metadata: Metadata = {
  title: "Next.js",
  description: "Generated by Next.js",
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

The `children` prop is where your page content renders. When visiting `/`, the content from `app/page.tsx` is passed as `children`.

## Creating a Custom Layout

Add shared UI elements like a header and footer:

```tsx
// app/layout.tsx
import type { Metadata } from "next"

export const metadata: Metadata = {
  title: "My App",
  description: "My Next.js application",
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <header
          style={{
            backgroundColor: "lightblue",
            padding: "1rem",
          }}
        >
          <p>Header</p>
        </header>
        <main>{children}</main>
        <footer
          style={{
            backgroundColor: "lightgray",
            padding: "1rem",
          }}
        >
          <p>Footer</p>
        </footer>
      </body>
    </html>
  )
}
```

### How It Renders

When you visit any page, the layout wraps the page content:

```
<html>
  <body>
    <header>Header</header>
    <main>
      <!-- Page content from children prop -->
      <h1>Welcome Home</h1>
    </main>
    <footer>Footer</footer>
  </body>
</html>
```

## Layout Rules

| Rule | Description |
|------|-------------|
| File Name | Must be `layout.tsx` or `layout.js` |
| Export | Must be a default export of a React component |
| Children | Must accept and render a `children` prop |
| Persistence | Layouts persist across navigations (do not re-render) |
| Nesting | Layouts can be nested inside other layouts |

## Layout vs Page

| Aspect | Layout | Page |
|--------|--------|------|
| Purpose | Shared UI wrapper | Route-specific content |
| File Name | `layout.tsx` | `page.tsx` |
| Re-renders | No (persists) | Yes (on navigation) |
| children Prop | Yes (receives page content) | No |
| Multiple per Route | Yes (nested) | One per route segment |

## Practical Example: Dashboard Layout

```tsx
// app/dashboard/layout.tsx
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <div className="dashboard-layout">
      <aside>Sidebar Navigation</aside>
      <section>{children}</section>
    </div>
  )
}
```

This layout applies a sidebar only to routes under `/dashboard`, while the root layout still provides the global header and footer.
